# End-to-End Insurance Risk Analytics

This repository contains the data engineering pipeline, reproducible environment setup, and initial exploratory analyses developed for an auto-insurance portfolio. The project leverages 18 months of historical claims data to evaluate data quality, isolate risk indicators, and establish a version-controlled workflow.

---

## Objectives

The primary analytical and operational goals for this phase of the project are:

* **Establish a Reproducible Development Workflow:** Set up a clean repo structure, an automated CI/CD linting pipeline, and isolated local version controls for all data states.
* **Formulate a Baseline Risk Profile:** Ingest historical policy, vehicle, and client information to understand overall portfolio profitability via core insurance performance metrics.
* **Assess and Remediate Data Quality:** Profile structural missingness, apply logical cleanups to implicit data types, and prepare stable data features for downstream mathematical modeling.

## Implementation Details

### Task 1: Environment Setup & Exploratory Data Analysis (EDA)

The modular analysis class (`src/insurance_eda.py`) handles statistical profiling and data hygiene tasks sequentially:

* **Descriptive Statistics & Baseline Profiling:** Extracts numerical summaries (`.describe().T`) and evaluates structural column formatting.
* **Data Quality & Schema Adjustment:** Screens for structural missingness (flagging parameters with over 50% blank fields). It typecasts dates (`TransactionMonth`, `VehicleIntroDate`) and normalizes messy string representations of currency objects into structured floats.
* **Domain-Specific Insurance Cleanup:** Maps implicit binary flags (`WrittenOff`, `Rebuilt`, `Converted`) to strict integer indicators and addresses demographic gaps via mode imputation. It eliminates rows missing critical registration keys (`make`, `Model`, `mmcode`) and optimizes categorical columns into standard memory-efficient category types.
* **Foundational Financial Evaluation:** Computes core insurance portfolio indicators across geographical distributions:

$$\text{Loss Ratio} = \frac{\text{TotalClaims}}{\text{TotalPremium}}$$


$$\text{Margin} = \text{TotalPremium} - \text{TotalClaims}$$


* **Statistical Visual Profiling:** Generates univariate feature distributions, constructs global correlation matrices, aggregates premium mapping variations across postal zones, and applies log-scaled box plots to evaluate risk-skewing outliers.

### Task 2: Data Version Control (DVC)

To ensure strict analytical reproducibility and maintain audit trails without bloating the Git history, the data directory is managed via DVC:

* **Isolated Pipeline Management:** The data directory contains a target `.gitignore` automatically generated by DVC, preventing large CSV inputs from being pushed to GitHub.
* **State Versioning:** Captures data pipeline progression by decoupling and versioning distinct tracking states (e.g., raw ingestion versus fully resolved and cast data outputs).
* **Local Storage Integration:** Links project revisions directly to independent target directory pointer remotes (`dvc remote add -d localstorage `), tracking data updates locally via secure pointer files (`.dvc`).

## Data Ingestion & Pipeline Reproduction

To recreate the exact data environment locally:

1. **Clone the Repo:**
```bash
git clone https://github.com/soliana-k/insurance-risk-analytics
cd insurance-risk-analytics

```


2. **Synchronize System Dependencies:**
```bash
pip install -r requirements.txt

```


3. **Pull Data Assets from Version Control Remote:**
Ensure your local DVC remote path is correctly configured to point to the directory setup, then fetch the versioned file binaries using:
```bash
dvc pull

```
